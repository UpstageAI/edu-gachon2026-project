# 프로젝트 이름: Food Compass

## 1. 프로젝트 소개

Food Compass는 지속되는 고물가 상황 속에서 장을 보기 전 "이거 지금 사도 될까?"라는 질문에 답해주는 LangGraph + RAG 기반 AI 에이전트입니다. 한국농수산식품유통공사(KAMIS) Open-API와 공공데이터포털(참가격) 데이터를 기반으로 실시간 가격을 조회·판정하고, 필요 시 더 저렴한 대체 품목까지 제안합니다.

- **한 줄 소개**: 장바구니 물가를 대신 판단해주는 AI 에이전트 — 단순 시세 조회를 넘어 "지금 사도 되는지"와 "대안"까지 제시합니다.
- **주요 사용자**: 한정된 생활비로 장을 보는 자취생 및 1인 가구, 나아가 다인 가구까지 활용 가능합니다.
- **프로젝트를 만들게 된 배경**: 실제 자취 경험에서 체감한 장보기 부담에서 출발했습니다. 농수산물 시세를 단순 조회하는 서비스는 있었지만, 사용자 상황에 맞춰 구매 여부와 대안을 판단해주는 서비스는 찾기 어려웠고, KAMIS·공공데이터포털을 통해 실시간 데이터 확보가 가능함을 확인한 뒤 프로젝트를 기획했습니다.
- **최종 결과물의 형태**: LangGraph 기반 에이전트 파이프라인 + FastAPI 백엔드(SSE 스트리밍) + Streamlit 챗봇 프론트엔드로 구성된 웹 서비스입니다.

---

## 2. 문제 정의

- **사용자가 겪는 불편함**: 자취생은 한정된 생활비 안에서 식재료를 구매해야 하는데, 농수산물은 계절·지역·판매처·수급 상황에 따라 가격 차이가 커서 지금 사는 것이 합리적인지 스스로 판단하기 어렵습니다. 또한 대량 구매보다 소량 구매가 많고 보관 기간·조리 수준이 개인마다 달라 획일적인 가격 정보만으로는 결정이 쉽지 않습니다.
- **기존 방식의 한계**: 시중 서비스는 농수산물 시세를 단순 조회하는 수준에 그쳐 있어, "지금 사도 되는지", "더 저렴한 대안은 없는지"에 대한 판단과 대안 제시까지 제공하지 않습니다.
- **왜 이 문제가 중요한지**: 지속되는 고물가 속에서 합리적인 구매 결정은 생활비 관리에 직접적인 영향을 주며, 이를 자동으로 도와주는 도구는 1인 가구뿐 아니라 다인 가구에도 실질적인 가치를 제공할 수 있습니다.

---

## 3. 문제 해결

- **핵심 아이디어**: 단순 시세 정보 제공을 넘어, 사용자의 질문 의도를 파악해 가격 판정 → 대체품 추천 → 근거 기반 답변까지 이어지는 판단형 에이전트를 구현했습니다.
- **AI/소프트웨어가 문제 해결에 사용된 방식**: LangGraph로 구성한 상태 기반 그래프가 사용자의 질문을 price / knowledge / hybrid / off-topic 4가지로 분류하고, 각 경로에 맞는 노드를 실행합니다. 벡터DB(ChromaDB)를 활용한 RAG로 대체 품목 검색과 제철·보관법 지식 답변을 근거 기반으로 생성하며, 환각 방지 하드 가드로 사실 왜곡을 걸러냅니다.
- **전체 동작 흐름**: 사용자가 지역·단위를 선택하고 질문을 입력하면 → Router가 의도를 분류하고 품목을 추출 → 2차 검증으로 오탐 문장을 필터링 → 가격/지식/비교 경로에 따라 KAMIS·참가격 데이터 조회 및 판정 → 비쌈 판정 시 RAG로 대체품 검색 → 최종 답변을 SSE로 실시간 스트리밍합니다.

---

## 4. 핵심 기능

- 자연어 의도 분류(Router): 질문을 가격/지식/가격+대체품/무관 질문 4가지로 분류하고 품목명 추출
- 2차 검증(Validation): 장난·롤플레잉 문장에 식품 키워드가 우연히 섞인 경우를 걸러내는 방어 로직
- 가격 판정: 최근 가격 등락률 기준 "비쌈/적정/쌈" 판정
- 단위 환산: kg/g, 개/단/포기 등 다양한 판매 단위를 원하는 기준으로 환산
- 원물 ↔ 가공식품 비교: 서로 다른 데이터 소스를 하나의 기준(예: 밥 1공기)으로 정규화해 비교
- 대체품 추천(RAG): 비쌈 판정 품목에 대해 같은 부류 내 벡터 유사도로 대체 품목 검색
- 지식 답변(RAG): 제철 정보·보관법 문서를 벡터DB에서 검색해 근거 기반으로 답변 생성
- 환각(hallucination) 방지 하드 가드: 답변에 사실 왜곡 감지 시 고정 템플릿으로 자동 폴백
- 실시간 스트리밍 응답: 진행 상태와 최종 답변을 SSE로 실시간 전달
- LLM 폴백: 주 모델 실패 시 백업 모델로 자동 재시도
- Streamlit 챗봇 프론트엔드: 지역/단위 선택 후 채팅 형태로 질의
