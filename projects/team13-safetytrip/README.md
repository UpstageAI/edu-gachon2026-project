# 재난안전 여행 가이드 AI 에이전트

## 1. 프로젝트 소개

> 재난안전 여행 가이드는 방문 지역·시기의 과거 재난 발생 통계를 자동 집계하고, 상황에 맞는 공식 행동요령을 대화형으로 안내하는 LangGraph 기반 AI 에이전트입니다. 근거가 불충분하면 답변을 만들지 않고 공식 대응기관으로 안내합니다.

- **한 줄 소개**: "이 지역, 이 시기에 여행 가는데 뭘 조심해야 하나요?"라고 물으면, 실제 재난문자 발령 이력과 공식 행동요령을 근거로 맞춤 안전 리포트를 제공하는 AI 에이전트
- **주요 사용자**: 국내 여행을 계획 중이거나 여행 중 재난문자를 받은 일반 사용자, 특히 노약자·어린이 등 취약계층 동반 여행객
- **프로젝트를 만들게 된 배경**: 행정안전부가 긴급재난문자 이력과 국민행동요령을 공공데이터로 제공하지만, 원자료 형태로 흩어져있어 일반 여행객이 그대로 활용하기 어렵다는 문제에서 출발
- **최종 결과물의 형태**: SSE 스트리밍 기반 REST API(FastAPI) + React 프론트엔드로 구성된 대화형 웹 서비스, GCE에 실배포

---

## 2. 문제 정의

> 여행객은 방문지의 실제 재난 발생 패턴이나 상황별 공식 대응 요령을 사전에 파악하기 어렵고, 재난 상황 발생 시에도 신뢰할 수 있는 즉각적인 안내를 받기 힘듭니다. 기존 공공데이터는 원자료 형태로만 제공되어 일반 사용자가 활용하기 어렵고, 검색으로 얻는 정보는 공식성이나 최신성이 보장되지 않습니다.

- **사용자가 겪는 불편함**: 여행지의 재난 위험도를 가늠할 데이터가 없고, 재난문자를 받아도 구체적으로 뭘 해야 하는지 모름
- **기존 방식의 한계**: 공공데이터포털의 원자료는 API 형태로만 제공되어 비개발자가 접근 불가능. 포털 검색 결과는 출처가 불분명하거나 지역·상황에 맞지 않는 일반론에 그침
- **왜 이 문제가 중요한지**: 안전 정보의 부정확성은 실제 인명 피해로 이어질 수 있어, 일반적인 챗봇 서비스보다 훨씬 높은 수준의 근거 기반 정확성이 요구됨

---

## 3. 문제 해결

> 사용자 질문에서 지역·시기·의도를 파싱한 뒤, 재난문자 이력 통계(SQL)와 공식 행동요령 벡터 검색(RAG)을 결합해 답변을 구성합니다. 검색된 근거가 질문에 실제로 답할 수 있는지 2단계(거리 임계값 + LLM 판정)로 검증하고, 근거가 불충분하면 답변 대신 공식 대응기관 연락처를 안내합니다. 이 판단의 정확도는 자체 평가셋으로 precision/recall/F1을 측정해 검증했습니다.

**핵심 아이디어**: "모르면 안다고 하지 않는다" — 통계와 행동요령 근거가 부족하면 LLM이 그럴듯한 답을 지어내지 않도록 2중 게이트로 차단

**AI/소프트웨어가 문제 해결에 사용된 방식**
- LangGraph로 파싱→통계/검색→게이트 판단→응답 생성의 다단계 흐름을 노드 단위로 관리
- Upstage Solar API(Chat/Embedding)로 자연어 파싱, 관련도 판정, 답변 생성, 벡터 검색 수행
- Supabase(Postgres+pgvector)에 재난문자 이력(53,631건), 행동요령(RAG, 751건), 대화 세션 상태를 통합 저장

**전체 동작 흐름**
```
사용자 질문
  → parse(지역/시기/의도/재난유형 추출)
  → stats(SQL 통계 집계) + retrieve(pgvector 벡터 검색)
  → gate 1차(거리 임계값) → gate 2차(LLM 적합성 판정)
  → respond(근거 인용 스트리밍 답변) 또는 escalate(공식 연락처 안내)
```

## 4. 핵심 기능

- 지역×시기 기반 재난 발생 통계 자동 집계 (표본 부족 시 상위 행정구역으로 자동 확대)
- 재난유형별 공식 행동요령 벡터 검색 및 근거 인용
- 2중 게이트(임계값+LLM 판정) 기반 에스컬레이션 — 근거 불충분 시 공식 대응기관 연락처 안내
- SSE 기반 실시간 스트리밍 답변
- 대화 세션 유지 — 후속 질문이 이전 턴의 지역·시기·동반자 맥락을 자동으로 이어받음
- LLM 3단 방어(timeout/재시도/강등)로 API 장애 시에도 안전하게 응답
- Langfuse+LangSmith 이중 트레이싱, Docker+GitHub Actions 기반 CI/CD 완전 자동화(빌드→마이그레이션→서버 배포)
