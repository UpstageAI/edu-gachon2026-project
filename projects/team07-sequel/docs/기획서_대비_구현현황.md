# 기획서 대비 구현 현황

> 기준일: 2026-07-10 · 브랜치: aiagent
> 원본 기획서: `docs/Sequel 프로젝트 기획서.pdf` (gitignore)

기획서(원안)와 현재 구현이 어디까지 같고, 무엇을 더 붙였고, 무엇이 남았고,
어떤 결정이 바뀌었는지 한 곳에 정리한다. **뒷단에 추가한 것은 전부 유지된다.**

---

## 0. 에이전트 흐름 (원안 → 현재)

```text
원안:  질문 → schema_linker → router → SQL생성(난이도4분기) → validator → executor → 포맷 → 응답
현재:  질문 → query_normalizer → schema_link → route → generate → validate → execute → format
                └ 신규 앞단              └ retriever 2종     └ 재생성 루프(최대 2회) ┘
```

정의 위치: [app/graph/builder.py](../app/graph/builder.py) · 상태: [app/graph/state.py](../app/graph/state.py)

## 1. 기획서에 없던 것 — 우리가 뒷단에 추가 (유지)

| 추가 | 기획서 | 현재 구현 | 파일 |
|---|---|---|---|
| **query_normalizer 노드** | 없음(흐름이 schema_linker부터) | 시간정규화(규칙)·키워드추출·후속병합·모호어 플래그 | `app/graph/nodes/query_normalizer.py` |
| **임베딩 schema_retriever** | "top-k 스키마 축소"만 | solar-embedding-2 양방향(테이블+컬럼)+elbow컷+FK보강, 인메모리 인덱스 | `app/tools/schema_retriever.py` |
| **value_retriever 폴백체인** | "셀 매칭"만 | exact→synonym→fuzzy→embedding, 카디널리티 인지·floor·ambiguous 되묻기 | `app/tools/value_retriever.py` |
| **few-shot 주입 훅** | RAG만 언급 | generator가 `state["fewshot"]` 주입(파인튜닝 대체). 벤치서 최대 정확도 레버 확인 | `app/graph/nodes/generator.py` |
| **질의분해(CTE) 가이드** | 없음 | 최상 난이도 프롬프트 단계분해 (`gen_decompose` 토글) | `app/core/prompts.py` |
| **dialect 인지 DB 접근** | Postgres만 | Supabase Postgres + AI Hub sqlite(평가용) 양쪽 | `app/core/db.py`, `app/repositories/*` |
| **벤치마크/평가 하네스** | 없음 | 모델 라우팅 실험·route_eval·난이도 역산·회귀셋 | `bench/*`, `tests/eval_linker.py` |
| **설정 단일화·튜닝 config화** | 없음 | pydantic-settings 단일 소스, 링킹 임계 전부 config | `app/core/settings.py` |
| CodeRabbit·API 명세 | 없음 | ko-KR 리뷰 설정, `docs/api.md`, `docs/difficulty_criteria.md` | 레포 루트, `docs/` |

## 2. 기획서대로 구현됨

FastAPI + SSE · LangGraph 흐름 · LiteLLM(solar 실연결) · 난이도 라우터 · injection 가드 ·
sqlglot validator · read-only executor(LIMIT·timeout) · 재생성 루프 · Supabase(읽기전용) · AI Hub 데이터.

## 4. 기획서와 달라진 결정

- **난이도 라우터 재해석**: 원안은 LLM 난이도 분류로 모델을 가른다. 벤치 결과
  ① 규칙기반 분류 정확도 35% < 다수결 45% ② 모델 간 성능차 작고 비용은 입력지배적
  → **난이도 라우팅의 이득이 작다**. 라우터는 유지하되(분류·가드는 계속), 실전은
  **전부 solar-pro2 고정**(`route_force_model`)으로 런칭 안정화. 검증된 난이도 값
  (하·중 mini / 상·최상 pro2)은 `settings.model_*`에 보존, 토글로 복귀 가능.
- **solar-pro3 제외**: 모든 난이도에서 pro2 이하(동일 단가) → 라우팅 후보 제거.
- **파인튜닝 불가·불필요** 확인 → few-shot(in-context)으로 대체.
- **Langfuse 연결 방식**: 수동 span 대신 litellm 네이티브 콜백으로 전 LLM 호출 자동 로깅.

## 5. 미해결 이슈

- **실 schema linker < 정적 값 샘플링 근사** (~15–20%p): value_retriever가 값이 아닌
  컬럼 개념 키워드("발행 연도" 등)를 셀 값에 억지 매칭 → 생성기 오도. 부분 방어
  (`_matchable` 가드) 적용, 잔여 노이즈 남음. 대안: 컬럼명 임베딩에 더 가까운 키워드는 스킵.
- **레이트리밋 500** 병렬 평가 시 잔존(4워커 ~9.7%) → 수치 디플레이트. 워커↓ 또는 딜레이 필요.
