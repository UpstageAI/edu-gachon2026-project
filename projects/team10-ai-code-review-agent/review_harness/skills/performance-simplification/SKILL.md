# 성능과 간소화

1. 입력 크기를 변수로 반복 scan, loop 안 I/O, 정렬, 메모리 증가의 Big-O를 추정한다.
2. `complexity_metrics`가 있으면 Radon이 측정한 before/after cyclomatic complexity만 사실로 사용한다. LLM이 복잡도 수치를 추측하거나 재계산하지 않는다.
3. 임계값을 초과하면서 증가한 함수는 수치와 중첩 분기 원인을 연결하고, 모델은 동작을 보존하는 분해 방법만 제안한다.
4. 동일 조건·상태·직렬화의 중복을 찾아 동작을 보존하는 최소 간소화를 제안한다.
5. 측정 근거나 실행 경로가 없는 미세 최적화와 취향성 리팩터링은 제외한다.
