# AI 코드 리뷰 에이전트

## 1. 프로젝트 소개

GitHub Pull Request의 CI 결과와 저장소 정책에 맞춰 리뷰 깊이를 자동으로 조절하고, RAG와 규칙 기반 하네스를 통해 근거있는 리뷰를 하는 AI 코드 리뷰 GitHub App입니다.

- **한 줄 소개**: 사용자의 정책을 반영하고, 근거를 제시하며 리뷰하는 AI 리뷰 에이전트
- **주요 사용자**: PR 리뷰 부담을 줄이고 싶은 개발팀과 코드 리뷰어
- **개발 배경**: LLM 리뷰는 쉽게 "말 많은 리뷰어"가 되어 근거 없는 지적과 비용을 늘린다. 우리는 리뷰의 성공을 *댓글 생성 여부*가 아니라 *정확성·근거·시간과 비용*으로 측정하고자 했다.
- **최종 결과물**: GitHub App(webhook) + FastAPI 서버. 대상 저장소에 App을 설치하면 PR/체크 이벤트에 반응해 자동으로 리뷰를 작성하고 GitHub Checks·댓글로 게시한다.

---

## 2. 문제 정의

- **사용자가 겪는 불편함**: 리뷰어는 단순 CI 실패(문법·린트·테스트)와 설계 수준의 문제를 같은 방식으로 처리하기 쉽고, 리뷰어마다 팀 정책 적용 기준이 달라진다.
- **기존 방식의 한계**: 큰 diff 전체를 LLM에 넣으면 응답 시간과 비용은 늘지만 정작 중요한 지적은 묻힌다. 또 LLM은 근거 없이 그럴듯한 지적(환각)을 만들어 신뢰를 떨어뜨린다.
- **왜 중요한가**: 리뷰 품질을 정량적으로 관리하지 못하면 AI 리뷰는 "많이 말하지만 믿기 어려운" 도구가 된다. 근거가 검증되고 비용이 통제된 리뷰여야 실제 병합 결정에 쓸 수 있다.

---

## 3. 문제 해결

CI 결과·정책·코드 신호를 규칙으로 분석해 상황에 맞는 리뷰 전략을 고른 뒤, 검증된 근거만 모델에 주입하고, 모델 결과를 다시 검증해 게시한다.

- **핵심 아이디어**: "무엇을 볼지"는 결정론적 규칙으로 좁히고, "그 안에서 판단"만 LLM에 맡긴다. 근거 선정과 결과 검증을 LLM 바깥에 두어 환각을 구조적으로 차단한다.
- **AI 활용 방식**: 하나의 Upstage Solar3 모델을 쓰되 경로에 따라 `reasoning_effort`만 low/medium/high로 다르게 적용한다. 리뷰 생성은 LiteLLM으로 호출한다.
- **전체 동작 흐름** (LangGraph StateGraph):
  1. **진입**: GitHub webhook 수신 → HMAC 서명 검증
  2. **라우팅**: CI 실패면 `실패 원인 리뷰`, 통과면 `정책 기반 표준 리뷰`, 사용자가 버튼을 누르면 `선택형 심층 리뷰`
  3. **컨텍스트**: diff 신호로 검토 절차(skill)·지식 카드·정책 유형을 선택(하네스), 관련 저장소 정책만 검색(RAG)
  4. **생성·검증**: Solar3 호출로 리뷰 생성 → 변경 파일·diff 줄·정책 출처·지식 카드 화이트리스트·중복·개수 상한을 검증
  5. **게시·저장**: inline/요약 댓글과 GitHub Check로 게시, 결과를 PostgreSQL에 저장, 진행 상황은 SSE로 스트리밍

## 4. 핵심 기능

- **CI 결과 기반 상황별 리뷰 라우팅**: 실패 원인 리뷰(low) / 정책 기반 표준 리뷰(medium) / 선택형 심층 리뷰(high)
- **규칙 기반 리뷰 하네스**: diff 신호로 검토 절차(skill)와 지식 카드를 선택해 오탐을 줄이고, 배치마다 최소한의 정책만 프롬프트에 주입
- **저장소 정책 RAG 검색**: 정책 문서를 chunk로 나눠 관련 정책만 검색(레거시 대비 정책 컨텍스트 약 95% 감축)
- **LLM 결과 검증**: 존재하지 않는 파일·줄·정책·지식 카드를 인용한 지적을 제거하고, 심각도 상한과 route별 개수 상한을 강제
- **리뷰 근거 명시**: 적용된 검토 절차·참조한 저장소 정책 문서·참고한 외부 지식 카드(OWASP·Google 등 공식 출처 링크)를 리뷰 댓글에 함께 표시
- **GitHub 통합**: GitHub Checks 연동, `심층 리뷰 실행` 버튼으로 선택형 심층 리뷰, inline·요약 댓글 자동 갱신
- **관측·배포**: Langfuse로 모델 호출(토큰·비용·지연) 추적, GitHub Actions로 GCP VM 자동 배포
